# 多维表引用数据表同步脚本 使用指南 v1.0

## 一、这个脚本是做什么的？

飞书多维表格中的「引用数据表」（侧边栏带⚡闪电图标的表）需要手动点击右键 →「同步数据」来刷新数据。

这个脚本可以**自动批量同步**所有指定的引用数据表，同步完成后在电脑右下角弹窗通知你。

---

## 二、使用前准备

### 2.1 安装 Node.js

1. 打开浏览器访问：https://nodejs.org
2. 点击绿色按钮 **Download Node.js**（推荐 LTS 版本）
3. 下载完成后双击安装包，一路点击「下一步」完成安装
4. 安装完成后，按 `Win + R` 键，输入 `cmd` 回车，打开命令行窗口
5. 输入以下命令验证是否安装成功：

```
node --version
npm --version
```

看到类似 `v20.x.x` 和 `10.x.x` 的版本号，说明安装成功。

### 2.2 安装 Google Chrome

如果电脑已安装 Chrome 浏览器，跳过此步。

1. 打开浏览器访问：https://www.google.cn/chrome
2. 点击「下载 Chrome」，双击安装包完成安装

### 2.3 登录飞书

在 Chrome 浏览器中打开飞书并登录你的账号，确保能正常访问多维表格。

## 三、配置脚本

### 3.1 找到配置文件

打开项目文件夹，找到 `config.json` 文件，用记事本或任意文本编辑器打开。

### 3.2 配置文件说明

```json
{
  "bitables": [
    {
      "name": "功能测试多维表",
      "url": "https://xxx.feishu.cn/base/xxx?table=xxx&view=xxx",
      "tables": [
        { "tableId": "tbl3EpCJ1W8mThIY", "name": "HZ006-考核汇总表（不含院外）" },
        { "tableId": "tblZKjmX65xfIhpE", "name": "MX002.1.1-25-26年健康元健可妥日流向" }
      ]
    }
  ],
  "chromeUserDataDir": "C:\\Users\\你的用户名\\AppData\\Local\\Google\\Chrome\\User Data",
  "chromeProfile": "Default",
  "syncTimeoutMs": 60000,
  "headless": false,
  "maxRetries": 2,
  "retryDelayMs": 3000
}
```

### 3.3 如何获取 tableId

1. 在 Chrome 中打开你的多维表格
2. 点击左侧导航栏中你要同步的表（带⚡闪电的表）
3. 看浏览器地址栏，URL 中 `table=` 后面的那串字符就是 tableId

示例：
```
https://j0eukrlohu.feishu.cn/base/FaKibNzneajMaSspa5Pc2ikQnzd?table=tbl3EpCJ1W8mThIY&view=vewxX14ebd
                                                                         ↑ 这就是 tableId
```

### 3.4 添加你要同步的表

在 `tables` 数组中添加你要同步的表，每行一个：

```json
"tables": [
  { "tableId": "第一个表的ID", "name": "第一个表的名称" },
  { "tableId": "第二个表的ID", "name": "第二个表的名称" },
  { "tableId": "第三个表的ID", "name": "第三个表的名称" }
]
```

> **注意**：每行末尾有逗号 `,`，但最后一行**不要**加逗号。

### 3.5 配置 Chrome 数据目录

修改 `chromeUserDataDir` 为你的实际路径：

```json
"chromeUserDataDir": "C:\\Users\\你的用户名\\AppData\\Local\\Google\\Chrome\\User Data"
```

> **注意**：路径中的反斜杠要写两个 `\\`，例如 `C:\\Users\\azhe\\...`

如果你不确定用户名是什么，按 `Win + R` 输入 `%USERNAME%` 回车即可看到。

## 四、运行脚本

### 4.1 首次运行（安装依赖）

1. 打开命令行窗口（按 `Win + R`，输入 `cmd` 回车）
2. 进入项目文件夹：

```
cd D:\develop_test\bitable_sync
```

> 将路径改为你实际的项目路径

3. 安装依赖（只需执行一次）：

```
npm install
```

等待安装完成，看到 `added xxx packages` 即可。

### 4.2 开始同步

```
npx tsx src/sync.ts
```

运行后会自动：
1. 启动 Chrome 浏览器
2. 打开多维表格
3. 逐个同步指定的引用数据表
4. 同步完成后弹出桌面通知
5. 自动关闭浏览器

### 4.3 运行结果

命令行会显示同步进度：

```

  [table] Clicked "HZ006-考核汇总表（不含院外）" in sidebar

  [sync] Sync completed for "HZ006-考核汇总表（不含院外）"
```

全部完成后显示汇总：

```

  ✅ 功能测试多维表 > HZ006-考核汇总表（不含院外）
```

同时电脑右下角会弹出通知窗口。

## 五、常见问题

### Q1: 提示「未登录」怎么办？

脚本启动的 Chrome 没有飞书登录态。请确保：
1. 你平时用的 Chrome 已登录飞书
2. `chromeUserDataDir` 和 `chromeProfile` 配置正确
3. 首次运行如果没登录，在弹出的浏览器窗口中登录飞书，然后重新运行脚本

### Q2: 某个表同步失败怎么办？

脚本会自动重试 2 次。如果仍然失败：
1. 检查该表是否在多维表格中仍然存在
2. 检查 tableId 是否正确
3. 检查网络连接是否正常
4. 下次运行时，已成功的表会自动跳过，只重试失败的表

### Q3: 想强制全部重新同步（不跳过已完成的）怎么办？

删除项目文件夹中的 `.sync-progress.json` 文件，然后重新运行脚本。

### Q4: 可以设置后台静默运行吗？

可以。将 `config.json` 中的 `"headless": false` 改为 `"headless": true`，浏览器将不可见地在后台运行。

### Q5: 同步超时怎么办？

如果某个表数据量很大，60 秒不够，可以调大 `syncTimeoutMs`，例如改为 120000（120 秒）：

```json
"syncTimeoutMs": 120000
```

### Q6: 运行时报错「找不到模块」怎么办？

运行 `npm install` 重新安装依赖即可。

## 六、配置文件完整参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `bitables` | 多维表格列表，每个包含 name、url、tables | 必填 |
| `chromeUserDataDir` | Chrome 用户数据目录路径 | 见上方示例 |
| `chromeProfile` | Chrome 配置文件名 | `"Default"` |
| `syncTimeoutMs` | 单个表同步超时时间（毫秒） | `60000`（60秒） |
| `headless` | 是否后台运行（true=不可见，false=可见） | `false` |
| `maxRetries` | 失败后最大重试次数 | `2` |
| `retryDelayMs` | 重试间隔时间（毫秒） | `3000`（3秒） |
