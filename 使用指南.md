# 多维表引用数据表同步脚本 使用指南（小白版）

## 1. 这东西是干嘛的？

一句话说明：
这个脚本会帮你自动打开飞书多维表，找到你指定的“引用数据表”（左侧通常带 ⚡ 图标），然后帮你点一次 **“同步数据”**。

适合的人：
- 你每天都要手动同步很多个引用数据表
- 你已经能在 Chrome 里正常打开飞书多维表

不适合的情况：
- 它不会帮你修改表里的数据
- 它不会创建新表
- 它只会按你在 `config.json` 里写好的表去同步

---

## 2. 这个项目实际是怎么工作的？

真正执行的入口文件是 `src/sync.ts`。

它每次运行时，会按下面的顺序做事：

1. 读取 `config.json`
2. 用你自己的 Chrome 登录信息启动浏览器
3. 打开你配置好的飞书多维表页面
4. 依次点击你配置好的目标表
5. 右键该表，点击“同步数据”
6. 如果失败，会自动重试
7. 把结果记录到 `.sync-progress.json`
8. 全部完成后弹出桌面通知

补充说明：
- 如果这次只成功了一部分，下一次运行时，已经成功的表会先跳过
- 只有当这次要处理的表全部成功后，进度文件才会自动清空

---

## 3. 使用前，你要准备什么？

只要 3 样东西：

1. 一台 Windows 电脑
2. 已安装 Google Chrome
3. 已安装 Node.js
4. 将整个压缩包下载后解压缩，并记住解压缩后的文件夹路径。

另外还要满足 1 个前提：
你必须已经在 Chrome 里登录了飞书，并且能手动打开目标多维表。

---

## 4. 第一次使用，按这个顺序做

### 第 1 步：安装 Node.js

1. 打开 [Node.js 官网](https://nodejs.org/)
2. 下载 **LTS** 版本
3. 双击安装包，一路点“下一步”
4. 安装完成后，按 `Win + R`
5. 输入 `cmd`，回车
6. 分别输入下面两行命令：

```bash
node --version
npm --version
```

如果能看到版本号，比如 `v20.x.x`、`10.x.x`，就说明装好了 ✅

### 第 2 步：确认 Chrome 能正常打开飞书

1. 打开 Chrome
2. 登录飞书
3. 手动打开一次你要同步的多维表
4. 确认左侧能看到那些要同步的引用数据表

这一步很重要。脚本本质上是在替你操作浏览器，所以你手动能打开，脚本才更容易成功。

### 第 3 步：打开项目文件夹

项目目录就是你现在这个文件夹，例如：

```bash
D:\develop_test\bitable_sync
```

### 第 4 步：安装项目依赖

按 `Win + R`，输入 `cmd`，回车，然后执行：

```bash
cd D:\develop_test\bitable_sync
npm install
```

说明：
- `npm install` 只需要第一次执行一次
- 如果以后删过 `node_modules`，或者换了电脑，再执行一次

### 第 5 步：修改 `config.json`

这是最关键的一步。

如果你不会改 JSON，也不用紧张。你可以理解成：
**只改双引号里的内容，其他括号、逗号、冒号尽量不要动。**

一个最常见的配置长这样：

```json
{
  "bitables": [
    {
      "name": "这里写这个多维表的备注名",
      "url": "这里粘贴这个多维表的网址",
      "tables": [
        { "tableId": "第一个表的ID", "name": "第一个表的备注名" },
        { "tableId": "第二个表的ID", "name": "第二个表的备注名" }
      ]
    }
  ],
  "chromeUserDataDir": "C:\\Users\\你的用户名\\AppData\\Local\\Google\\Chrome\\User Data",
  "chromeProfile": "Default",
  "syncTimeoutMs": 60000,
  "headless": false,
  "maxRetries": 2,
  "retryDelayMs": 3000
}
```

下面只讲你最需要改的几个地方。

#### 你要改 1：`name`

这个只是备注名，给你自己看。

比如：

```json
"name": "销售日报多维表"
```

#### 你要改 2：`url`

这里填这个多维表的网址。

做法：
1. 在 Chrome 里打开目标多维表
2. 复制浏览器地址栏的网址
3. 粘贴到 `url` 这里

比如：

```json
"url": "https://xxx.feishu.cn/base/xxx?table=xxx&view=xxx"
```

#### 你要改 3：`tables`

这里写“具体要同步哪些表”。

每个表需要两个信息：
- `tableId`：表的 ID
- `name`：你自己看得懂的备注名

示例：

```json
"tables": [
  { "tableId": "tbl3EpCJ1W8mThIY", "name": "销售汇总" },
  { "tableId": "tblZKjmX65xfIhpE", "name": "门店流向" }
]
```

#### 你要改 4：怎么找 `tableId`

1. 在 Chrome 里打开目标多维表
2. 点击左侧你要同步的那张表
3. 看地址栏
4. 找到 `table=` 与 `&view` 之间的那串字符

示例：

```text
https://xxx.feishu.cn/base/abc123?table=tbl3EpCJ1W8mThIY&view=vew123
                                     ↑ 这段就是 tableId
```

#### 你要改 5：`chromeUserDataDir`

这里填你电脑上 Chrome 用户数据目录。

常见格式：

```json
"chromeUserDataDir": "C:\\Users\\你的用户名\\AppData\\Local\\Google\\Chrome\\User Data"
```

注意两点：
- `你的用户名` 要换成你电脑自己的用户名
- 反斜杠必须写成 `\\`，不能只写一个 `\`

如果你不知道用户名，也可以直接去资源管理器里打开：

```text
C:\Users
```

看看你自己的文件夹名是什么。
如果在内网环境，可能有多个用户名文件夹，例如：`姓名拼音.公司名`

#### 你要改 6：`chromeProfile`

大多数人保持默认就行：

```json
"chromeProfile": "Default"
```

如果你平时用的是 Chrome 的其他用户，例如“Profile 1”，那这里就写对应的文件夹名。

### 第 6 步：保存 `config.json`

改完后直接保存即可。

如果你不确定自己有没有写对，先检查 3 件事：

1. 每个双引号是不是成对出现
2. 最后一行后面不要多写逗号
3. `tableId` 和 `url` 是不是从真实页面复制来的

## 5. 怎么运行？

打开命令行窗口，执行：

```bash
cd D:\develop_test\bitable_sync
npx tsx src/sync.ts
```

运行后，你会看到这些现象，都是正常的：

- 会自动打开 Chrome
- 会自动进入飞书多维表
- 浏览器会自己点击左侧表名
- 会自己右键并点击“同步数据”
- 结束后会弹出桌面通知
- 浏览器会自动关闭

建议：
运行过程中，尽量不要手动去点这个自动打开的浏览器窗口，避免打断脚本操作。

## 7. 如何让 Windows 定时自动运行？

如果你希望每天、每周自动跑一次，可以用 Windows 自带的 **任务计划程序**。

先说结论：
- 小白最推荐用“任务计划程序”
- 第一次设置前，先手动跑通一次脚本
- 如果你想让它安静地在后台跑，建议把 `config.json` 里的 `"headless": false` 改成 `"headless": true`

### 7.1 先做 2 个准备

1. 先确认下面这条命令你已经手动运行成功过：

```bash
cd D:\develop_test\bitable_sync
npx tsx src/sync.ts
```

2. 如果你打算定时后台跑，建议先把 `config.json` 改成：

```json
"headless": true
```

说明：
- `true`：后台运行，平时更适合定时任务
- `false`：运行时会弹出浏览器，适合排错

### 7.2 用任务计划程序创建定时任务（推荐）

#### 第 1 步：打开任务计划程序

1. 按 `Win + S`
2. 搜索：`任务计划程序`
3. 点击打开

#### 第 2 步：创建基本任务

1. 在右侧点击 **“创建基本任务...”**
2. 名称填：`飞书多维表同步`
3. 描述可以填：`按计划自动运行飞书多维表同步脚本`
4. 点击“下一步”

#### 第 3 步：选择多久运行一次

你可以按自己的需求选：
- `每天`
- `每周`
- `每月`

对大多数人来说，通常选 `每天` 或 `每周` 就够了。

然后继续设置：
- 开始时间
- 每隔几天/几周执行一次

设好后点击“下一步”。

#### 第 4 步：选择“启动程序”

在“操作”这一步，选择：

```text
启动程序
```

然后点击“下一步”。

#### 第 5 步：填写程序和命令

按下面这样填：

**程序或脚本：**

```text
C:\Windows\System32\cmd.exe
```

**添加参数（可选）：**

```text
/c cd /d D:\develop_test\bitable_sync && npx tsx src/sync.ts
```

**起始于（可选）：**

```text
D:\develop_test\bitable_sync
```

填完后点击“下一步”。

#### 第 6 步：完成创建

检查一遍信息没问题后，点击“完成”。

这时，一个定时任务就建好了。

### 7.3 建好后，建议再改 3 个地方

在任务计划程序里找到你刚创建的任务，右键它，点“属性”。

建议检查下面几项：

#### 常规

推荐优先选择：

```text
仅当用户登录时运行
```

原因：
这个脚本要读取你当前 Windows 用户下的 Chrome 登录信息。对小白来说，这个选项通常更稳，更少出意外。

#### 条件

如果你用的是笔记本，而且希望不插电也能执行，可以看看有没有这项：

```text
只有在计算机使用交流电源时才启动此任务
```

如果你希望用电池时也能跑，可以取消勾选。

如果你希望电脑睡眠时也能被叫醒执行，还可以留意是否有：

```text
唤醒计算机以运行此任务
```

#### 设置

建议勾上类似下面的选项：

```text
错过计划后尽快运行任务
```

这样即使电脑刚好关机或睡眠，之后也更容易补跑。

### 7.4 先手动测试一次

不要等到第二天才发现没跑。

建好任务后，建议立刻测试：

1. 在任务计划程序中找到这个任务
2. 右键它
3. 点击 **“运行”**
4. 看看脚本是否真的正常执行

如果能正常跑，说明定时任务基本配好了 ✅

### 7.5 一个推荐配置

如果你只是想“每天早上自动同步一次”，可以这样配：

- 触发器：每天
- 时间：`09:00`
- 运行方式：`仅当用户登录时运行`
- `config.json`：`"headless": true`

这样通常最省心。

## 9. `.sync-progress.json` 是什么？

这是脚本自动生成的“进度记录文件”，你一般不用手动改它。

它的作用是：
- 记录哪些表已经同步成功
- 如果上次只完成了一部分，下次运行时可以跳过已经成功的表

### 想强制全部重新跑一遍怎么办？

删除项目目录里的 `.sync-progress.json`，再重新运行脚本。

## 10. 常用参数，小白先记这 4 个就够了

### `headless`

```json
"headless": false
```

意思：
- `false`：看得见浏览器，推荐新手先用这个
- `true`：后台运行，看不见浏览器

### `syncTimeoutMs`

```json
"syncTimeoutMs": 60000
```

意思：单个表最多等多久，单位是毫秒。

常见换算：
- `60000` = 60 秒
- `120000` = 120 秒

如果某些表数据很多、经常超时，可以把它调大一点。

### `maxRetries`

```json
"maxRetries": 2
```

意思：失败后，最多再试几次。

### `retryDelayMs`

```json
"retryDelayMs": 3000
```

意思：两次重试之间等多久。

## 11. 常见问题

### Q1：运行后跳到了登录页，怎么办？

先检查下面 3 件事：

1. 你平时用的 Chrome 里是否已经登录飞书
2. `chromeUserDataDir` 是否写对了
3. `chromeProfile` 是否写对了

如果弹出的浏览器里还没登录，就先登录一次，再重新运行脚本。

### Q2：提示某个表找不到，怎么办？

通常检查这几项：

1. `tableId` 是否复制错了
2. 这张表是否还存在
3. 你是否还有这个表的访问权限
4. 这张表是否真的在左侧边栏里能看到

### Q3：某个表一直超时，怎么办？

把 `config.json` 里的超时时间调大，例如：

```json
"syncTimeoutMs": 120000
```

### Q4：为什么有些表这次没跑？

因为脚本会读取 `.sync-progress.json`，自动跳过之前已经成功的表。

如果你希望不跳过，直接删掉 `.sync-progress.json` 再运行。

### Q5：我能不能让它后台运行？

可以，把：

```json
"headless": false
```

改成：

```json
"headless": true
```

但如果你是第一次用，建议先保持 `false`，因为看得见浏览器更容易排查问题。

### Q6：提示找不到模块，怎么办？

回到项目目录，重新执行：

```bash
npm install
```

### Q7：定时任务里点“运行”没反应，或者提示找不到 `npx`，怎么办？

先检查两件事：

1. 你手动在命令行里执行下面命令是否能成功

```bash
cd D:\develop_test\bitable_sync
npx tsx src/sync.ts
```

2. 任务计划程序里的“程序或脚本”和“添加参数”是否照着上面填写

如果还是不行，可以把“程序或脚本”改成 Node.js 常见安装路径里的 `npx.cmd`：

```text
C:\Program Files\nodejs\npx.cmd
```

然后把“添加参数”改成：

```text
tsx src/sync.ts
```

“起始于（可选）”保持为：

```text
D:\develop_test\bitable_sync
```

### Q8：定时任务手动点“运行”可以，但到时间没有自动跑，怎么办？

通常检查这几项：

1. 电脑到点时是不是关机或睡眠了
2. 任务触发时间是不是设错了
3. 是否选择了“仅当用户登录时运行”，但到点时你并没有登录系统
4. 笔记本是否因为“只有在交流电源下运行”而被拦住了
5. 是否勾选了“错过计划后尽快运行任务”

## 12. 多个多维表怎么配？

如果你不止一个多维表要同步，就在 `bitables` 里继续加一组。

你可以把下面这一整段复制一份：

```json
{
  "name": "第二个多维表备注名",
  "url": "第二个多维表的网址",
  "tables": [
    { "tableId": "第二个多维表里的第一个表ID", "name": "表1备注名" }
  ]
}
```

放到 `bitables` 数组里，和第一个对象并列即可，注意多个多维表配置项之间用英文逗号分隔。

---

## 13. 最后给小白的建议 🙂

第一次配置时，先不要一口气加很多表。

建议顺序：

1. 先只配 1 个多维表
2. 先只配 1 到 2 个引用数据表
3. 跑通一次后，再慢慢增加

这样最容易排错，也最不容易把问题搞复杂。
